# 专利技术交底书

## 一、 发明创造的名称
一种基于解耦点刚性配额与优先级排程的双层规划系统及方法

---

## 二、 本发明的技术领域
本发明属于企业级主生产计划、高级计划与排程（APS）、供应链运筹决策、物料需求计划（MRP）以及运筹优化决策系统技术领域。

## 三、 背景技术与现有技术的缺陷
在现代半导体制造（包含晶圆中道制造与后道封装测试）、高端通信设备制造、精密汽车电子制造等超大规模离散工业中，生产运营体系极其庞大且瞬息万变。生产系统通常包含数万种原料、在制品（WIP）和产成品，且包含极其错综复杂的工艺路径（Routing）、设备机台约束以及交期承诺要求。
为了应对这一高度复杂的巨系统，工业界在系统架构上普遍采用横向和纵向解耦的双层规划体系：
1. **战术计划层（ITP，Tactical Planning）**：主要负责在中远期（如未来 1-3 个月，以周或月为时间桶 Time Buckets）进行滚动的供需消纳，做宏观的物料与产能配额分配（Allocation），用来指导战略物料采购、设备投资和宏观生产大盘平衡。
2. **运营计划层（IOP，Operational Planning）**：负责在近期（如未来 1-7 天，以天或小时为时间桶）进行高精度的微观排程（Scheduling/Dispatching）和物料齐套关联（Pegging），处理波动的真实客户加急订单与生产异常。

然而，现有的双层计划系统（如 SAP APO、Blue Yonder、o9 等）在实际纵向协同和数据传递中，存在严重的行业瓶颈与缺陷：
在大规模跨企业边界的外包制造或多工厂协同网络中（例如品牌商主导、多家自建工厂及外部代工厂/ODM协同制造的模式），由于各执行主体（自建工厂、不同的外部代工厂）之间存在天然的利益博弈，这一跨层级协同失效问题尤为致命。品牌商在战术层做出了明确的产品家族或大客户群体的配额分配决策（Allocation），用以平衡不同代工厂的产能水位和保证核心大客的全球战略履约。然而，在微观 IOP 层级执行日常排产时，各代工厂为了提升自身的“订单准时交付率（OTIF）”或机台利用率，一旦突然涌入大量高优先级的紧急加急订单，微观排程引擎会凭借“绝对高优先级”将品牌商提供的通用客供料（如主芯片、高端屏幕）或代工厂间的共用产能抢占一空。这导致战术层预留给核心客户的刚性配额被强行“踩踏”，宏观战略平衡彻底失效，核心大客无法履约，带来巨额违约损失。

* **第一，跨层级协同失效导致的“非理性高优订单抢占与资源劫掠”。**
  企业通常在战术层做出了明确的产品家族或客户群体的配额分配决策（Allocation），用以平衡不同市场区域的呆滞库存风险和保证核心客户的战略履约（例如：本周将通用芯片封测产能的 60% 分配给核心家族A，40% 分配给高利润家族B）。然而，在微观 IOP 层级执行日常排程时，一旦突然涌入大量高优先级的紧急散客订单，微观排程引擎为了提升局部的“订单准时交付率”，会凭借“绝对高优先级”将通用的原材料或机台产能抢占一空。这导致战术层预留给核心家族A的刚性配额被强行“踩踏”，宏观战略平衡彻底失效，核心大客无法履约，带来巨额的违约罚款。
* **第二，计算时效性冲突与计划震荡（Nervousness）。**
  为了防御上述“踩踏”，若在微观排产中加入复杂的跨层级配额硬性校验，传统的数据库事务控制或基于行级锁的校验机制在百万级订单规模下面临极其灾难性的计算延迟和时空死锁（Deadlocks）。引擎无法在短时间内完成消纳校验，导致 What-If 沙箱模拟时间长达数小时，根本无法满足制造现场秒级调度响应的要求。频繁的全局重算又会导致计划系统频繁震荡。

因此，如何在保障战术大盘战略配额（Allocation）刚性约束的前提下，赋予运营层极高排产灵活度，并在百万级订单规模下实现毫秒级响应的跨层级协同配额保护与无锁时空前移置换，是高级计划与排程领域极具挑战的瓶颈问题。

## 四、 本发明要解决的技术问题
本发明的目的在于提供一种基于解耦点刚性配额与优先级排程的双层规划系统及方法，用以解决传统工业排程中跨层级分配失效、高优订单盲目抢占导致大盘战略失衡的技术痛点，实现高并发下毫秒级响应的无锁时空前移置换与大盘配额硬性保护。

## 五、 本发明的技术方案（核心创新点及具体实现）
为了实现上述目的，本发明提供的技术交底方案包括以下四个层面的核心实现：

### 1. 战术大盘 Allocation 滚动规划与下传
战术计划（ITP）模块将未来的客户预测和战略合同汇总至周或月度的粗时间跨度（Time Buckets）中，通过运筹大盘滚动求解，计算并输出各个产品家族在各个时间跨度内的资源分配总量（Allocation）。
在数学模型上，上层 ITP 规划以最大化战略大客承诺额和最小化宏观供需差距为目标建立线性规划模型：
$$ \min \sum_{f \in F} \sum_{T \in \mathcal{T}} ( W_f \cdot (Demand_{f, T} - Supply_{f, T}) + C_f \cdot Leftover_{f, T} ) $$
其中 $F$ 为产品家族集合，$\mathcal{T}$ 为战术计划时段集合。$Demand_{f, T}$ 为预测需求，$Supply_{f, T}$ 为实际排产供应，$Leftover_{f, T}$ 为宏观呆滞期末结存，$W_f$ 为未满足惩罚权重，$C_f$ 为持有库存惩罚。

### 2. 解耦点 Allotment 刚性屏障转化机制（Decoupling Allotment）
本发明通过解耦点配额转换模块，在多级物料清单 BOM 的关键库存缓冲点处锚定物料解耦点（Decoupling Point）。系统自动将 ITP 层的时段 Allocation，沿着 BOM 树向下投影展开拆解并聚合成各产品家族在解耦点处的刚性配额上限（Allotment）。
具体转化公式如下：对于解耦点物料 $D$，其产品家族 $f$ 在运营时段 $t$（对应战术时段 $T$）内的刚性配额 $Allotment_{D, f, t}$ 为：
$$ Allotment_{D, f, t} = \sum_{p \in Family(f)} Allocation_{p, T} \times BOM\_Ratio(p, D) \times \gamma_{t} $$
其中，$Family(f)$ 表示产品家族 $f$ 包含的最终成品 SKU 集合；$BOM\_Ratio(p, D)$ 表示成品 $p$ 在其多级 BOM 展开到解耦点 $D$ 时的单位消耗量；$\gamma_{t}$ 为运营时段 $t$ 在战术时段 $T$ 内的时间分布权重系数，满足所有属于 $T$ 的运营时段 $t$ 的 $\gamma_{t}$ 累加和为 1。Allotment 作为硬性屏障输入运营排程引擎，任何微观操作均不可逾越该上限。

### 3. 运营层 64位二进制复合优先级与交期倒排消纳
运营计划（IOP）模块接收真实客户订单，为避免传统结构体多层条件判断导致的 CPU 分支预测失败，系统使用 64 位无符号整数（uint64_t）进行高低位位段拼装以表示复合优先级，拼装公式为：
`encoded = ((uint64_t)committed_bit << 62) | ((uint64_t)tier_bits << 60) | ((uint64_t)due_bits << 44) | ((uint64_t)pri_bits << 28) | (uint64_t)revenue_bits`
其中比特分配及业务含义如下：
* `committed_bit`（占1位，第62位）：代表承诺状态位，已承诺订单为0，未承诺订单为1（已承诺优于未承诺）。
* `tier_bits`（占2位，第60-61位）：代表客户级别段，战略客户对应0，大客户对应1，普通客户对应2，散客对应3（数值越小优先级越高）。
* `due_bits`（占16位，第44-59位）：代表订单相对计划基准日的交期天数。交期越早，数值越小，优先级越高。
* `pri_bits`（占16位，第28-43位）：代表ERP导入的原始优先级。数值越小，优先级越高。
* `revenue_bits`（占28位，第0-27位）：代表金额反算段。计算公式为：$M - \min(M, Revenue)$，其中 $M = 268435455$。订单金额 Revenue 越大，反算段越小，优先级越高。

IOP 排程求解器按照上述优先级从小到大的全局升序，依次对订单执行交期倒排拉动消纳：从交期出发向左倒排提前期，在解耦点处净消纳其对应的可用配额余额（可用配额 = 初始配额 - 已分配配额）。若可用配额归零，则强制闭锁该时段 of 刚性配额分配入口，并将订单向更迟的时段顺延。

### 4. 同家族配额内时空无锁置换机制（OTP Preemption）
当同一产品家族在特定时段的刚性配额已被占满，而营业窗口又涌入了该家族更高优先级的战略加急订单时，时空置换求解模块启动：
* **步骤一**：定位同一时段（如 Day t）内已排产的、同属于该产品家族且优先级较低的普通订单集合；
* **步骤二**：反向回滚（Rollback）该低优先级订单在当前时段占用的配额与产能；
* **步骤三**：执行前移平移，将低优先级订单的排产时间向右平移（往更迟的 Day t+1 或 Day t+2），在更迟时段的同一家族配额池内为其重新排产；
* **步骤四**：将原时段腾退出的配额空间分配给战略加急订单，保障其准时排产。
置换过程严格受限在同家族的 Allotment 额度池内，绝对禁止跨产品家族抢占或劫掠其他家族的空闲配额。

### 5. 底层高性能计算架构与数据库直通管道
运营计划模块采用面向数据设计（DOD）的内存管理排布，将反映物理实体的**对象数据模型（ODM）**与反映控制策略的**控制数据模型（CDM）**高度紧凑地集成在同一个连续的 `PartSiteRecord` 结构体中，使 CPU 缓存利用率达到最大化：
```cpp
struct PartSiteRecord {
    uint32_t part_id;                // ODM: 物理物料 ID
    std::string part_code;           // ODM: 物理物料编码
    double on_hand;                  // ODM: 实际物理现有量
    uint32_t low_level_code;         // ODM: 静态 LLC 拓扑码
    double lead_time;                // ODM: 基准前置期
    std::string mrp_rule;            // CDM: 需求消纳与展开法则
    std::string sourcing_policy;     // CDM: 替代料与分摊策略
    std::string planning_calendar;   // CDM: 工厂生产日历
    double run_rate;                 // CDM: 前置期自适应拉伸系数
};
```
在整个计划求解迭代周期内，系统将所有 `PartSiteRecord` 实例存放在 `std::vector<PartSiteRecord>` 的物理连续内存中，完全禁止任何动态堆内存申请。求解器在进行 IOP 级联消纳时，可通过单次 Cacheline 加载同时获得物理物料状态与 CDM 计划参数，极大减少了内存寻址延迟。消纳计算完毕后，结果通过无锁的 DuckDB 内存 `Appender` 管道，在同进程内以二进制块形式直接流式写入列式沙箱表中，完成高速直通灌库。

## 六、 本发明的技术效果及创新点
1. **物理阻断跨家族抢占**：
   通过在 BOM 解耦点处设定刚性上限配额，在物理上把通用的物料划分为各个产品家族的独立配额池，从根本上杜绝了跨产品家族的非理性高优踩踏，确保了战术大盘战略 Allocation 比例的刚性执行。
2. **毫秒级高并发无锁置换**：
   64位二进制复合优先级排序极大地加快了比对速度，DOD 内存排布和无锁前移置换机制避免了传统行级锁带来的死锁和计算延迟，在百万级规模下达到毫秒级求解。
3. **保证宏观大盘与微观敏捷平衡**：
   既保障了战术大盘分配的战略比例，又赋予了运营排程同一家族内的高低优先级弹性置换能力，实现了资源配置的全局最优与现场敏捷排产的和谐统一。

### 1. 复合优先级拼装与 std::sort 排序实施例
设系统包含以下 4 笔待排产订单，相对计算基准日为 Day 0，大值常量 M = 268435455：

  `committed_bit` = 0 (位移62位) ➔ `0x0`
  `tier_bits` = 2 (位移60位) ➔ `0x2000000000000000`
  `due_bits` = 6 (位移44位) ➔ `0x600000000000`

## 八、 与先前及后续专利的关联关系

本专利作为整个 IPC 系统层级的“宏微双层协调大脑”，起到了级联消纳与刚性屏障管控的核心作用，它与后续其他协同专利构成了紧密的底层算法与数据协同网络：
1. **与专利 7（一种基于嵌入式列式内存数据库物理沙箱的多版本 What-If 协同规划系统及方法）的关联**：
   本专利的双层规划在进行宏观 ITP 预测冲销与微观 IOP 优先级排程时，所有计算均在专利 7 所克隆隔离的进程内沙箱物理文件（如 `sandbox_Si.db`）中进行。IOP 消纳成功后，其生成的计划订单通过专利 7 的无锁 `Appender` 管道直接写入该沙箱表中，确保 What-If 并发分析的物理数据隔离与一致性。
2. **与专利 8（一种基于物料依赖 LLC 拓扑分级的大大规模 MRP 并行消纳调度系统及方法）的关联**：
   本专利在计算解耦点 Allotment 刚性配额时，需要沿着多级 BOM 展开。此时，系统调用专利 8 的 LLC 拓扑分级并行计算框架，在各个无依赖的层级内部使用 OpenMP 多线程对产品家族配额进行并行累加展开，从而将战术 ITP 层的 Allocation 极速转化为 IOP 层的 Allotment 屏障。
3. **与专利 9（一种基于原子级乐观时间戳锁的跨区域/渠道虚拟安全库存池化共享系统及方法）的关联**：
   在 IOP 层级按照 64 位优先级倒排消纳时，如果遇到共享替代料，引擎会调用专利 9 提供的基于 CAS 乐观锁协议的虚拟安全库存池扣减算子，在不引发互斥锁竞争的前提下，完成多渠道/多产品家族对共享替代料库存的并发安全占配。
4. **与专利 10（一种基于线程局部零堆分配回滚栈与时空网格的有限能力排产系统及方法）的关联**：
   当本专利的 IOP 求解器在同家族内执行 OTP 抢占平移置换（步骤二中回滚低优先级订单在当前时段的占用）时，所有的产能与物料扣减动作均记录在专利 10 描述的线程局部零堆分配回退栈中。一旦置换路线探索失败，回退栈以 $O(1)$ 速率瞬间原位还原 Day t 的产能日历，不引发堆内存申请。
5. **与专利 11（一种基于数据模型同构双螺旋与自反馈控制的抗熵计划调节系统及方法）的关联**：
   当车间实际执行出现交期偏差时，专利 11 启动自反馈抗熵环路，动态调整本专利 `PartSiteRecord` 结构体中的 CDM 控制参数（如修改 `run_rate` 或重算 `sourcing_policy`），再通过本专利的双层管道进行重新 Netting 纠偏，形成控制闭环。
