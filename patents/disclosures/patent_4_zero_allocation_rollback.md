# 专利技术交底书

## 一、 发明创造的名称
一种面向级联排产的零分配内存事务性回滚系统及方法

---

## 二、 本发明的技术领域
本发明属于离散制造高级计划与排程（APS）、多级物料需求计划（MRP）、内存数据库、软件事务内存（STM）以及高性能计算引擎底层算法技术领域。

---

## 三、 背景技术与现有技术的缺陷
在离散制造供应链计划与排程中（如可承诺量 CTP 多级级联排产），计算引擎为了判定一笔新客户订单是否可以按时交付，必须高频进行大量的“尝试性扣减与状态回退”计算。对于一个包含 15 至 20 层 BOM（物料清单）的复杂离散工业成品（如高端超算服务器、精密晶圆测试设备），当营业窗口接收一笔大额客户订单并执行 CTP（Capable-to-Promise）排产时，计算引擎需要从最终成品开始，沿着多级 BOM 拓扑树一路向下进行深度优先递归消纳，对上百个子物料节点在特定时间点的可用量（ATP）执行尝试性扣减，同时还需要扣减生产工序所占用的关键设备产能（Capacity）。

这种多级级联扣减具有极强的物理“齐套性（Kitting Constraints）”约束（即木桶效应）：只要最底层或中间任何一级的子物料在相应时间点发生短缺，或者某台关键机台产能耗尽，整条成品的级联排产路径即宣告失败。一旦路径失效，为了防止宝贵的在库资源与产能被错误锁定，系统必须立即将在这笔订单处理过程中、于各个上层父物料和设备资源上已经作出的全部尝试性扣减和占用彻底撤销（Rollback），使全系统的在库量和产能完全恢复到排产前的干净初始状态。

然而，现有的计划排程系统在处理这种高频级联排产的事务性回退时，存在严重的技术缺陷与瓶颈：
* **第一，频繁申请堆内存对象导致高频垃圾回收（GC）停顿。**
  为了在失败时能够顺利撤销，现有的 Java/C# 排产引擎每次向下递归时，都需要动态创建一个“分配快照对象”或“历史记录备份”来暂存历史值。对于包含数百万笔订单的大规模排产，这会产生数以亿计的临时垃圾对象，导致 JVM 堆内存频繁发生垃圾回收（Garbage Collection）停顿，极大地拖慢了计算性能，系统无法达到毫秒级响应。
* **第二，物理数据库 I/O 与行锁冲突开销昂贵。**
  如果将尝试性扣减和事务控制直接交给底层的关系型数据库，高频的磁盘物理写盘（WAL日志）与行级事务锁竞争将导致排程时间被拉长至数小时，完全无法满足工业现场 What-If 秒级沙箱推演的要求。

因此，如何在多层级 BOM 的复杂物理约束下，实现一种零动态内存申请、零磁盘 I/O 开销、在物理共享内存中极速原位还原的级联排产事务性回滚机制，是高级排程底层引擎设计极具挑战性的技术痛点。

---

## 四、 本发明要解决的技术问题
本发明的目的在于提供一种面向级联排产的零分配内存事务性回滚系统及方法，用以解决多级物料与产能级联扣减中频繁动态内存申请、垃圾回收停顿剧烈以及行锁等待高昂的技术痛点，实现基于物理共享内存、微秒级收敛的原位无损回滚。

---

## 五、 本发明的技术方案（核心创新点及具体实现）
为了实现上述目的，本发明提供的技术交底方案包括以下四个层面的核心实现：

### 1. 共享物理内存与零分配（Zero-Allocation）追踪缓冲区
系统在启动点火阶段，在物理连续内存中提前申请固定大小的全局分配追踪缓冲区（Allocation Trace Buffer）。该缓冲区完全禁止在排产消纳迭代中执行运行时的动态内存申请（如 C++ 的 new/delete 或 Java 的堆内存实例化），从而在底层消除了内存碎片与垃圾回收停顿。
为了适应供应链中多工厂替代寻源、LLC解耦点刚性配额控制、采购比例限制及动态物料替代等复杂业务场景，缓冲区中除了记录基础的物料和产能扣减外，还包含了动态替代追踪、刚性配额追踪以及采购比例均衡追踪结构体。
在 C++ 计算核心中，数据结构定义为扁平的连续数组：
物料扣减追踪结构体：
```cpp
struct MaterialTrace {
    uint32_t part_index; // 连续物理内存中的物料 SKU 索引
    uint16_t day_idx;    // 时间切片天数索引
    double qty;          // 临时尝试扣减的数量
};
```
产能占用追踪结构体：
```cpp
struct CapacityTrace {
    uint32_t resource_id; // 机台设备资源物理索引
    uint16_t day_idx;     // 时间切片天数索引
    double qty;           // 临时占用的产能工时（小时）
};
```
动态替代追踪结构体（记录替代比例与不完全替代限制）：
```cpp
struct SubstitutionTrace {
    uint32_t original_part_index;   // 原始物料 SKU 索引
    uint32_t substitute_part_index; // 替代物料 SKU 索引
    uint16_t day_idx;               // 时间切片天数索引
    double original_qty;            // 原始需求数量
    double substitute_qty;          // 实际扣减的替代数量
    double substitution_ratio;      // 动态替代比例因子 β (可随数量或日期动态变化)
};
```
刚性配额占用追踪结构体（LLC解耦点分销渠道或大客户的 rigid quota）：
```cpp
struct AllotmentTrace {
    uint32_t allotment_pool_id;     // 刚性配额池物理索引
    uint16_t day_idx;               // 时间切片天数索引
    double qty;                     // 占用配额数量
};
```
采购/排产比例均衡追踪结构体（用于监督多工厂采购比例约束）：
```cpp
struct PurchasingRatioTrace {
    uint32_t vendor_group_id;       // 采购/排产均衡组物理索引
    uint16_t day_idx;               // 时间切片天数索引
    uint32_t vendor_index;          // 具体供应商/工厂索引
    double qty;                     // 临时分配的采购/生产量
};
```
全局追踪缓冲区管理器：
数据结构设计：连续内存分配追踪缓冲区（Allocation Trace Buffer）

本发明在共享物理内存中提前声明并申请固定大小的连续追踪缓冲区，定义以下扁平数组：
1) `temp_allocations`：物料尝试性扣减痕迹数组（MaterialTrace 扁平结构体数组）
2) `temp_capacity_allocations`：设备产能尝试性扣减痕迹数组（CapacityTrace 扁平结构体数组）
3) `temp_substitution_allocations`：不完全物料替代尝试性扣减痕迹数组（SubstitutionTrace 扁平结构体数组）
4) `temp_allotment_allocations`：解耦点渠道刚性配额尝试性扣减痕迹数组（AllotmentTrace 扁平结构体数组）
5) `temp_purchasing_allocations`：供应商/工厂采购比例均衡尝试性扣减痕迹数组（PurchasingRatioTrace 扁平结构体数组）

系统在点火启动时，针对上述各数组使用 `reserve()` 操作提前预分配物理上限空间，在计算运行期绝对禁止任何动态堆内存扩展。

### 2. 深度优先搜索（DFS）级联扣减的原位追踪算子
求解器沿着多级 BOM 拓扑结构向下执行深度优先搜索。在对特定物料节点或设备产能执行“尝试性扣减”时，系统不备份整个数据库，也不复制物料表。
系统仅在对物理共享内存中的在库量数组或产能数组执行扣减的瞬间，将该节点的物理数组索引（`part_index`）、扣减数量（`qty`）以及扣减发生的时间天数索引（`day_idx`）作为一个微小记录体（Record）追加写入对应的追踪向量中。
共享物理内存中的库存和产能同时执行原位减法扣减：
$$ OnHand[part\_index, day\_idx] = OnHand[part\_index, day\_idx] - qty $$
同时将此记录追加至 `temp_allocations`。

### 3. 物理逆向原位回滚事务算子（Reverse Rollback Engine）
一旦 DFS 递归深度检查判定在某级节点（例如 Level 15）发生缺料或产能不足，整条订单交付链失效，则逆向回滚引擎立即启动：
* **反向遍历**：系统以从后向前（Reverse Order）的顺序读取追踪缓冲区中的记录。
* **原位加回**：根据记录的物理数组索引、天数索引和数量，在共享内存对应的在库数组与产能数组格点上执行物理数值原位加回：
  $$ OnHand[part\_index, day\_idx] = OnHand[part\_index, day\_idx] + qty $$
  $$ Capacity[resource\_id, day\_idx] = Capacity[resource\_id, day\_idx] + qty $$
* **重置计数器**：通过调用 `clear()` 将物料和产能追踪向量的内部元素计数器（`Size`/`Pointer`）直接清零。由于追踪向量已预先分配物理内存，清空计数器仅移动边界指针，不释放任何堆内存，在 1 微秒内即可完成事务性还原。

### 4. 物理确权同步管道
在 DFS 级联消纳路径最终判定成功时，系统将分配追踪缓冲区中记录的全部尝试性扣减，批量转换为最终的确权扣减结果，并通过高速直通管道灌入底层的 DuckDB 列式物理库中，流式同步给前端控制塔展示。

## 六... 本发明的技术效果及创新点
1. **极速状态无损还原**：
   通过反向遍历和原位加回，在共享内存中以低于 10 微秒的计算开销完成了复杂的级联回滚，无需任何磁盘 I/O，确保了秒级 What-If 沙箱推演的闭环响应。
2. **零动态内存分配与无 GC 停顿**：
   整个计算生命周期没有任何动态内存申请操作，消除了传统 APS 中对象 high 频生命周期变化导致的 GC 抖动，最大化单核 CPU 吞吐。
3. **最小化行锁和线程竞争**：
   通过原位逆向覆写，使并发的尝试性扣减都在只读或共享状态下有序进行，避免了传统事务在回退时的大面积锁竞争。
4. **支持多重业务规则的回滚校验**：
   在回滚追踪中深度融入了刚性配额池（Allotment Quota Pool）、多工厂采购比例分配状态及动态替代因子约束，确保任何一种业务约束被触发违反时，系统都能在微秒级内完整回滚全部级联变更，保证了复杂多中心排产业务逻辑的一致性。

## 八、 与先前及后续专利的关联关系

本专利作为“级联物料与产能需求深度优先搜索零分配回滚引擎”，是整个计划引擎实现微秒级事务控制的底层支柱，它与本系统的其他专利形成了互锁的底层协同网络：
1. **与专利 7（一种基于嵌入式列式内存数据库物理沙箱的多版本 What-If 协同规划系统及方法）的关联**：
   本专利的在库现有量 OnHand 数组和产能 Capacity 数组均直接映射在专利 7 所创建的沙箱数据库物理文件（如 `sandbox_Si.db`）对应的进程内存空间中。本专利在进行深度优先（DFS）级联扣减时，直接在该内存空间中原位改写。若扣减成功，则通过专利 7 的无锁 Appender 管道批量确权写入沙箱库表；若失败，则在沙箱内存空间执行反向原位加回，零开销地恢复干净初始状态，避免了脏数据污染沙箱。
2. **与专利 8（一种基于物料依赖 LLC 拓扑分级的大大规模 MRP 并行消纳调度系统及方法）的关联**：
   当专利 8 以 LLC 分级并行机制沿着有向超图自上而下消纳物料需求时，在遇到需要齐套性（Kitting）检查的多层级子 BOM（例如包含 15 层以上的复杂工艺组件）时，系统启动本专利。本专利利用预分配的全局追踪缓冲区 `AllocationTraceBuffer`，记录该分支内各 LLC 级别的尝试性物料扣减动作。如果最底层采购件或中间半成品缺失，本专利逆向遍历缓冲区原位退回，协同专利 8 实现了绝对干净且零锁竞争的层内并行 Netting。
3. **与专利 9（一种基于原子级乐观时间戳锁的跨区域/渠道虚拟安全库存池化共享系统及方法）的关联**：
   本专利在向下级联搜索、对替代件执行尝试性扣减时，系统会调用专利 9 的乐观 CAS 扣减算子，在不加互斥锁的情况下预占虚拟安全库存。一旦本专利的后续计算判定齐套失败触发回退，系统除了执行原位加回外，会调用专利 9 对应的 CAS 原子释放操作，将占用的虚拟安全库存无锁归还到共享池中。
4. **与专利 10（一种基于线程局部零堆分配回滚栈与时空网格的有限能力排产系统及方法）的关联**：
   本专利聚焦于**多级物料 BOM 网络级联需求的原位回滚**；而专利 10 聚焦于**微观车间有限产能时空网格排产（带换型时间 setup_time 计算）的线程局部栈回滚**。两者在 APS 求解器中构成了协同关系：当本专利在进行多级物料展开检查时，在遇到需要排产的制造工序时，会调用专利 10 的有限能力回滚栈记录该机台的产能扣减和换模开销。若齐套失败，两者以相同路径分别回滚物料现有量与设备日历网格，确保多维约束的绝对正确性。
5. **与专利 11（一种基于数据模型同构双螺旋与自反馈控制的抗熵计划调节系统及方法）的关联**：
   当物理探针捕获车间物理异常（如某批次物料报废，导致实际在库降为 0）时，专利 11 计算残差并启动自反馈闭环。为了修正计划，专利 11 重新调用本专利的 DFS 原位消纳算子在沙箱中对替代方案进行快速试算。一旦试算路线不满足约束，立即利用本专利的回退引擎微秒级恢复，协助系统寻找新的最优抗熵校准点。
6. **与专利 2（一种基于联副产品分级分流消纳的配产控制系统及方法）的关联**：
   当专利 2 执行联副产品分级分流计算并调用 Projected Gradient 进行多级投产规划时，若生成的生产决策在下级装配中遇到物料或刚性配额缺口，会即时触发本专利的零分配事务回退，撤销已生成的虚拟产出及相应的惩罚状态，使专利 2 能够安全地回溯并调整步长或者迭代方向。
7. **与专利 3（一种基于时域并行前缀和分治的同节点ATP供需平衡消纳系统及方法）的关联**：
   当销售端高频并发询单并调用专利 3 的时域并行段树进行 ATP 承诺时，如果在段树前缀扫描中判定存在局部缺口，则会调用本专利的 DFS 回退引擎，将尝试性分配的配额及调拨转换状态在微秒级内全部撤销，并释放被占用的调拨转换率状态，确保时域二叉段树数据的洁净度。
