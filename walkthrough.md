## 🛠️ Changes Made（修改内容）

### 1. 修复第九定律（全景图）大一统场景空白与不运动 Bug 
* **缺陷原因一**：第九场景 `buildPanoramicScene()` 试图循环遍历并渲染系统维度坐标轴 `dimensions.forEach` 和实体物流运输节点 `nodesData.forEach`，但由于全局变量 `dimensions` 和 `nodesData` 未定义，引发 `ReferenceError` 导致渲染引擎崩溃，表现为“点击第九场景一片空白”。
* **缺陷原因二**：在第九场景的动画更新循环中，通过 `scene.traverse` 循环寻找 `THREE.Points` 时，代码里使用了一个未定义的全局变量 `particleSystem` (`child !== particleSystem`) 进行过滤，导致动画在第一帧直接抛出 `ReferenceError: particleSystem is not defined`，导致整个 3D 渲染彻底锁死停转（表现为“全景图静止不动”）。
* **修复方案**：
  * 在全局作用域中完整声明了五维数据容器实体集 `nodesData`（含 Plant A 总装厂、Plant B 部件厂、DC East、DC South 分销仓，以及 Tier-1/Tier-2 级供应商实体坐标）。
  * 声明了 VCP 正交空间中的 8 个基础维度坐标轴 `dimensions` 的三维向量及显影配色。
  * 将第九场景的物料粒子系统提升为全局变量 `panoFlowParticlesPoints`，在 `animate()` 循环中直接通过此变量名精确、高速地更新粒子轨迹，彻底移除了慢速的 `scene.traverse` 并破除了未定义 `particleSystem` 的崩溃点。
  * 现在全景场景完美且流畅地运动起来：旋转的双螺旋中枢、发光的八维控制轴线、外围 Allotment 网格防护边界、以及实时的多级供需货运粒子流正常闪烁流转。

### 2. 目的论场景（第一定律）- 部门“矢量对消”与业务拉扯实体化
* **物理模型投影**：价值链中有效做功 $V$ 取决于部门意志向量的对齐。当缺乏全局意志（Purpose）牵引时，销售、采购、生产、财务四部门的偏好向量发生冲突对消。
* **三维视觉显影**：
  * 引入 4 根不同颜色的部门意志矢量箭头：红色 🔴 代表销售，紫色 🟣 代表采购，黄色 🟡 代表生产，绿色 🟢 代表财务。
  * **目的场引力关闭**：4 根矢量箭头朝不同方向无序旋转（Brownian chaotic motion），且货运粒子散乱漂移，直观模拟部门在割裂状态下的“矢量对消（Vector Cancellation）”。
  * **目的场引力开启**：4 根矢量箭头瞬间顺滑旋转对齐至统一的旋转轨道（Vector Alignment），货运粒子有序向心收拢，模拟全局协同做功最大化。
* **业务悬浮标签**：绑定部门实体，动态输出拔河话术。关闭引力场时，标签显示各部门本位诉求（如销售：“我要丰富备货！”；生产：“我要连续大批次跑产！”；财务：“我要库存占用归零！”），展示真实的管理拉扯。

### 3. 机制论场景（第五定律）- 阿罗不可能定理决策死锁与对话框显影
* **物理模型投影**：将传统的 Allotment 战术配额重构为**阿罗不可能定理**物理同构机制——多目标偏好在无中枢算法独裁时闭环死锁（矢量对消）。
* **三维视觉显影**：
  * 以发光的中央计算中枢（Brain）为核心，外围均匀分布 4 个部门实体节点及各自的决策向量。
  * **委员会民主协商开启**：4 个部门向量各自旋转扭动，互相碰撞冲突。连接线变红，并在中央摩擦碰撞出大量红色闪烁火花粒子，模拟因投票循环陷入的组织高耗能内耗死锁（Arrow Deadlock）。
  * **算法逻辑独裁开启**：中央中枢（Brain）射出亮青色光束，强制锁死所有部门的自由度，迫使所有决策矢量绝对方向对齐，火花瞬间消失，Q（内耗）跌至谷底，OTIF 与 ROIC 强力拉升。
* **部门对话气泡**：在委员会模式下，4个部门节点头顶弹出本位博弈话术（“销售：我要产满备货！”；“采购：我要大批量折扣！”等），中枢显示“【阿罗死锁】决策陷入死循环”。算法独裁模式下，对话自动切换为“【绝对对齐】执行全局最优排产”。

### 4. 方案论（第三定律）与能力论（第四定律）- 新增“物理断裂与红色剪切”动态反馈
为了丰富中间几个定律的视觉表达，使其对价值链的解构更加形象：
* **第三定律 (双螺旋本体)**：
  * 对 D-A 双核咬合滑块加入了动力学阻尼计算。
  * 当双螺旋向左右偏移时，连结两者的数据因果链 (Pegging rungs) 不仅随之向两侧物理拉扯，而且在咬合度低于 30% 时瞬间发生**“动态崩断（Shatter & Snap）”**：连结线收缩为 0.1倍长度、由青色化为高频闪烁的红色警告光栅，预示着数据和算法断开，系统产生巨大呆滞内耗。
* **第四定律 (单脑奇点)**：
  * 连结三层物理镜像平面的中央单脑处理器激光柱（Cylinder Singularity），增加了动态偏序约束反馈。
  * 关闭“单脑对齐”且平面的利益偏离角 θ 拉大时，激光柱立刻**由发光的青色变红，收紧缩水为高频抖动红光丝线**，代表中枢立法权丧失、系统三层平面（物理执行、商业契约、财务记账）断裂错位。

### 5. 全面覆盖的 HTML 悬浮业务标识卡片 (HTML Float Labels System)
为了彻底破除原系统“过于抽象、脱离业务”的问题，引入了基于 CSS 3D 投影的动态悬浮业务标识卡片系统：
* **第一定律 (目的论)**：标注 **“🎯 全局 ROIC 吸引子 (目标: 35%)”** 及各部门立场。
* **第二定律 (本质论)**：标注 **“🧠 碳基人工 Excel 排程”**（脑力破产，O(N!) 约束爆炸）与 **“⚡ 硅基 C++ 求解器”**（5分钟极限消纳50万订单）。
* **第三定律 (方案论)**：标注双螺旋本体的物理成分 **“数据容器 D”**（地点/BOM/物料/需求/供给）与 **“算法算符 A”**（因果/置换/配额）。
* **第四定律 (能力论)**：标注三层格网流形为 **“💼 商业契约与规划层”**、**“⚙️ 物理执行与物料层”**、**“💰 财务核算与价值层”**。
* **第六定律 (路径论)**：标注底座为 **“📶 物理车间 IoT 传感器阵列”** 并支持 HZ 参数反馈。
* **第七定律 (动力论)**：标注做功夹角并计算实时做功效率 **`cosθ`**。

### 6. 实时状态卡片 (VCP State Card) 动态加载
* **设计目的**：在屏幕中顶部（#top-bar 下方）植入高档毛玻璃效果的 `vcp-state-card`。
* **实现方式**：随着滑块 and 开关状态的切换，卡片实时变色和刷新文字。例如：
    * ⚠️ **矢量对消**（红色边框）：无全局意志，系统合力为零，做功 $V$ 降为零！
    * 🎯 **矢量对齐**（绿色边框）：销售 🔴、采购 🟣、生产 🟡、财务 🟢 向量协同对齐。
    * 🛑 **协商死锁**（红色边框）：偏好投票循环死锁，向量对消并爆发摩擦废热。
    * ⚡ **算法独裁**（青色边框）：硅基大脑强制收敛全局唯一最优路径。
