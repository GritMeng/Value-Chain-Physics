## 🏛️ 1. 状态跃迁的数据本体定义 (Ontological Definition)

在 `[[17_数据本体双螺旋结构]]` 体系中，**状态跃迁 (State Transition)** 并不是一个孤立的静态数据实体，而是驱动数据在 **运营数据模型 (ODM)**、**控制数据模型 (CDM)** 与 **决策数据模型 (DDM)** 之间流转与突变的**离散动力学相变算符**。

状态跃迁的本质，是**“物理做功”在商业确权与资金流上的晚期绑定 (Late Binding)**。在数据库结构中，它由离散的状态字段（Status Fields）、供需分配连结关系（Pegging Relations）以及财务凭证账（Ledger Entries）共同承载。

```
 物理奇点事件 (Physical Singularity)
 [车间物料消耗 / 库房出库发运]
              │
              ▼
   ┌────────────────────────────────────────────────────────┐
   │         跨维映射流形 1对3 正交投影 (1-to-3 Projection)     │
   └──────┬─────────────────────┬─────────────────────┬─────┘
          │                     │                     │
          ▼                     ▼                     ▼
 ┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
 │  物流投影 (ODM)  │  │  商流投影 (CDM)  │  │  资金投影 (DDM)  │
 │  物料消耗/半成品  │  │  物权交割与 SWAP   │  │  资产折旧与 WIP  │
 └──────────────────┘  └──────────────────┘  └──────────────────┘
```

## 🏛️ 2. 全量数据模型中的四大状态跃迁链路

结合 `[[ipc_data_model_all.js]]`，系统的状态跃迁在底层数据表结构中具体表达为以下四条确定性链路：

### 1. 独立需求生命周期状态机 (Demand Lifecycle Transition)
*   **物理动作**：合同签订、销售预测滚动、物料实际出库发运。
*   **数据模型表达**：
    - `ipc_sales_order_line` (销售订单行) 与 `ipc_forecast` (预测) 的生命周期状态由关联表 `ipc_demand_status` 管理。
    - **状态突变字段**：通过 `ipc_demand_status.control_class` 与其配置的 `forecast_against_date_rule` 规训需求的时间边界。
    - **预测冲减跃迁**：当实际销售订单在时间轴上冲减预测明细 `ipc_forecast_detail` 时，预测记录的状态自动由“未消耗 (Unconsumed)”跃迁为“已消耗 (Consumed)”。
    - **时空截止退化**：一旦物理时间越过 `ipc_part_site.demand_tf_intervals` 设定的时间栅栏，未冲减的预测记录状态强制突变为“失效 (Expired)”，从算符的物理消纳域中被彻底抹除。

### 2. 供需分配的“确权”与状态锁定 (Pegging Allocation & Rights Transition)
*   **物理动作**：库存到位、生产工单开工、货运在途追踪。
*   **数据模型表达**：
    - 状态跃迁的轨迹被实时记录在 `ipc_supply_assignment` (供需分配关系表) 中，它是连接需求（Demand）与供给（Supply）的拓扑纽带。
    - **库存状态突变**：当 `ipc_onhand` (在手库存) 被分配给特定销售订单时，其 `inventory_type` (库存类型) 属性由“自由可用 (Free)”强制覆写为“合同锁定 (Reserved)”。
    - **跃迁分类**：
      - **软性预留 (Soft Pegging)**：关联至预测需求，处于“柔性非排他状态”，可被高优先级订单动态置换。
      - **刚性确权 (Hard Pegging / Locked)**：关联至刚性销售合同或关键试产 WBS 节点，在物理与逻辑上被双重死锁，进入“不可侵占状态”。

### 3. 时间轴上的动能实例化 (Kinetic Instantiation)
*   **物理动作**：系统滚动重算，诊断出未来某时空点 $(Site, Part, T)$ 的供给赤字（缺料）。
*   **数据模型表达**：
    - 计划引擎根据 CDM 中设定的获取规则 `SourceRule`，在时间轴的未来坐标上无中生有地实例化（Instantiate）出 `ipc_planned_order` (计划订单表) 记录。
    - **跃迁本质**：生成计划订单将系统在未来的状态从“缺口状态 (Deficit)”强行跃迁至“动态配平状态 (Balanced)”。
    - **物理约束写入**：`ipc_planned_order` 写入了经由提前期折算得到的 `request_start_date` (计划开工期) 与 `due_date` (期望完工期)，直接驱动下游车间工单与供应商采买日历的相变响应。

### 4. 业财一体化与价值投影 (Financial Settlement & Value Projection)
*   **物理动作**：发运过账、工单产出报工。
*   **数据模型表达**：
    - 在物理跃迁完成的瞬间，`ipc_supply_assignment` 对应的供需关系被强制终止，并顺着跨维映射流形，向 `ipc_financial_ledger` (财务明细总账) 进行自动过账。
    - **财务状态跃迁**：原料资产转化为 WIP (在制品) 或 COGS (销货成本)；同时根据 `ave_sales_price` 抓取结算单价，确认营业收入并核算由于延迟交付产生的 penalty cost (罚款损失)。

## 🏛️ 3. 状态跃迁的沙盘隔离与审计机制 (Sandbox & Audit)

为了防止计算推演污染真实的生产系统，状态跃迁支持在内存沙盘中进行安全模拟：
*   **情景隔离表 (`ipc_collab_scenario`)**：用户可创建隔离的模拟沙盘，在沙盘中自由微调 CDM 中的控制参数（如缩短采购提前期，或调整分配比例）。
*   **审计日志表 (`ipc_collab_audit_log`)**：引擎在沙盘内高频计算时，每一次状态跃迁（PlannedOrder 新增、Onhand 状态反写）都被以增量日志的形式写入审计日志，作为碳基审批前的决策审计凭证。
